# Integration icon / brand assets

Home Assistant shows integration logos from the central
[`home-assistant/brands`](https://github.com/home-assistant/brands) repository
(served via `brands.home-assistant.io`), **not** from this repository. Until our
icon is merged there, Home Assistant shows an "icon not available" placeholder on
the integration page. This is purely cosmetic — the integration works fine
without it.

## Assets (ready to submit)

In [`custom_integrations/cbus/`](custom_integrations/cbus/):

- `icon.png` — 256×256, transparent, rounded badge
- `icon@2x.png` — 512×512 (hDPI)
- `logo.png` — 512×512 (square logo; brands also accepts a wordmark)

## How to make the icon appear in Home Assistant

Submit the assets to `home-assistant/brands`:

1. Fork <https://github.com/home-assistant/brands>.
2. Copy this folder's `custom_integrations/cbus/` into the fork's
   `custom_integrations/cbus/` (keep `icon.png` + `icon@2x.png`; `logo.png`
   optional).
3. Open a pull request. Brands requires: square, trimmed, transparent
   background, PNG, and the hDPI `@2x` variant — all satisfied here.
4. Once merged, the icon appears automatically in any Home Assistant that has
   this integration installed (no integration update needed).

The source icon is generated by [`make_icon.py`](make_icon.py) (Pillow) — run it
to regenerate or tweak the design.
